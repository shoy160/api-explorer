# API Explorer 开发路线图

## 1. 依赖分析

### 1.1 地基模块（必须优先完成）

| 模块 | 说明 | 依赖关系 |
|-----|------|---------|
| **类型定义** | OpenAPI 类型、全局类型 | 无 |
| **工具函数** | 通用工具、常量定义 | 类型定义 |
| **状态管理** | Auth、Settings、Favorites、History | 工具函数、类型定义 |
| **HTTP 客户端** | Axios 封装、拦截器 | 状态管理（Token） |
| **OpenAPI 解析服务** | 文档解析、Schema 处理 | 类型定义、工具函数 |

### 1.2 核心业务模块（MVP 必备）

| 模块 | 说明 | 依赖关系 |
|-----|------|---------|
| **布局组件** | Header、Sidebar、Layout | 状态管理（Settings） |
| **API 列表展示** | 分组列表、接口搜索 | OpenAPI 解析服务 |
| **API 详情展示** | 请求参数、响应参数 | OpenAPI 解析服务 |
| **接口调试** | 在线请求、响应展示 | HTTP 客户端、状态管理 |

### 1.3 增强功能模块（可延后）

| 模块 | 说明 | 依赖关系 |
|-----|------|---------|
| **代码生成** | 多语言代码生成 | OpenAPI 解析服务 |
| **收藏功能** | 接口收藏、取消收藏 | 状态管理（Favorites） |
| **请求历史** | 历史记录、快速重试 | 状态管理（History） |
| **主题切换** | 亮色/暗色主题 | 状态管理（Settings） |
| **多语言** | 中英文切换 | 状态管理（Settings） |
| **一键复制** | 接口地址、代码复制 | UI 组件 |

## 2. 里程碑规划

### 2.1 Milestone 1: MVP（第 1-2 周）

**目标**：完成核心功能闭环，实现基本的 API 文档浏览和调试能力

| 序号 | 任务 | 描述 | 状态 |
|-----|------|------|------|
| 1 | 项目初始化 | 配置 Vite + React + TypeScript + Tailwind CSS | [ ] |
| 2 | 类型定义 | 创建 OpenAPI 类型定义文件 | [ ] |
| 3 | 工具函数 | 编写通用工具函数和常量 | [ ] |
| 4 | 状态管理 | 实现 Auth、Settings store | [ ] |
| 5 | HTTP 客户端 | 封装 Axios，支持 Token 拦截 | [ ] |
| 6 | OpenAPI 解析服务 | 集成 swagger-parser，解析文档 | [ ] |
| 7 | Mock 数据 | 创建示例 OpenAPI 文档用于测试 | [ ] |
| 8 | 布局组件 | 实现 Header、Sidebar、主布局 | [ ] |
| 9 | API 列表 | 展示接口分组和列表 | [ ] |
| 10 | API 详情 | 展示请求参数和响应结构 | [ ] |
| 11 | 接口调试 | 实现在线请求和响应展示 | [ ] |

**完成标准**：
- 能够加载并解析 OpenAPI 文档
- 能够展示接口列表和详情
- 能够发起测试请求并查看响应

---

### 2.2 Milestone 2: 完整版（第 3-4 周）

**目标**：完成所有核心功能，达到生产可用状态

| 序号 | 任务 | 描述 | 状态 |
|-----|------|------|------|
| 12 | 代码生成 | 实现 cURL、JavaScript、Python、Java、Golang 代码生成 | [ ] |
| 13 | 收藏功能 | 实现接口收藏和取消收藏 | [ ] |
| 14 | 请求历史 | 记录请求历史，支持快速重试 | [ ] |
| 15 | 主题切换 | 实现亮色/暗色主题切换 | [ ] |
| 16 | 多语言 | 实现中英文界面切换 | [ ] |
| 17 | 一键复制 | 接口地址、请求示例、响应示例复制功能 | [ ] |
| 18 | 接口搜索 | 实现接口名称和路径搜索 | [ ] |
| 19 | 设置页面 | 实现全局 Token 设置、主题设置 | [ ] |
| 20 | 移动端适配 | 响应式布局适配手机和平板 | [ ] |

**完成标准**：
- 支持代码生成（5 种语言）
- 支持收藏和历史记录
- 支持主题和语言切换
- 支持复制功能
- 移动端可正常使用

### 2.3 Milestone 3: 增强版（第 5-6 周）

**目标**：优化体验，添加高级功能

| 序号 | 任务 | 描述 | 状态 |
|-----|------|------|------|
| 21 | 性能优化 | 虚拟滚动、懒加载、缓存策略 | [ ] |

| 27 | 构建优化 | 代码分割、Tree Shaking | [ ] |
| 28 | 部署文档 | 编写部署指南和使用说明 | [ ] |

- 大型文档加载流畅
- Mock 数据生成功能可用
- 代码覆盖率达标
- 可快速部署

## 3. 开发顺序建议

### 3.1 第一阶段（基础层）

```
1. 项目初始化 → 2. 类型定义 → 3. 工具函数 → 4. 状态管理 → 5. HTTP 客户端
```

### 3.2 第二阶段（服务层）

```
6. OpenAPI 解析服务 → 7. Mock 数据
```

### 3.3 第三阶段（UI 组件）

```
8. 布局组件 → 9. API 列表 → 10. API 详情 → 11. 接口调试
```

### 3.4 第四阶段（增强功能）

```
12. 代码生成 → 13. 收藏功能 → 14. 请求历史 → 15. 主题切换 → 16. 多语言
```

### 3.5 第五阶段（完善与优化）

```
17. 一键复制 → 18. 接口搜索 → 19. 设置页面 → 20. 移动端适配
```

## 6. 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|-----|-------|------|---------|
| OpenAPI 规范兼容性 | 高 | 无法解析部分文档 | 使用成熟的解析库，做好异常处理 |
| 大型文档性能问题 | 中 | 页面卡顿 | 实现虚拟滚动和懒加载 |
| 移动端适配问题 | 中 | 用户体验差 | 优先使用响应式设计 |
| 代码生成准确性 | 中 | 生成代码有误 | 编写测试用例验证 |
