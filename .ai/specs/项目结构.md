## 4. 状态管理结构

### 4.1 Store 模块职责

| 模块 | 状态内容 | 操作方法 |
|-----|---------|---------|
| `api` | 接口列表、当前选中接口 | 加载文档、选择接口 |
| `auth` | 全局 Token | 设置/获取 Token |
| `favorites` | 收藏列表 | 添加/移除收藏 |
| `history` | 请求历史记录 | 添加/清除历史 |
| `settings` | 主题、语言 | 更新设置 |

## 5. 数据流架构

```
用户操作 → 组件 → Hook → Store → Service → API/Storage
                                              ↓
                                        数据返回 → Store → Hook → 组件 → UI 更新
```

### 数据流向说明

1. **用户操作**：用户在 UI 上进行操作（点击、输入等）
2. **组件层**：组件接收用户操作并调用相应的 Hook
3. **Hook 层**：处理业务逻辑，调用 Store 方法
4. **Store 层**：管理状态，调用 Service 层
5. **Service 层**：执行实际的业务逻辑（API 请求、本地存储等）
6. **数据返回**：Service 返回数据到 Store
7. **状态更新**：Store 更新状态并触发组件重新渲染
8. **UI 更新**：组件根据新状态更新界面
