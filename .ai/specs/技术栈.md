# API Explorer 技术栈方案

## 1. 技术选型总览

基于项目需求分析（OpenAPI 文档解析、API 调试、响应式布局、多语言支持等），推荐以下技术栈组合：

| 分类 | 技术 | 版本 | 选型理由 |
|-----|------|------|---------|
| **前端框架** | React | 18.x | 生态成熟，社区活跃，适合构建复杂的单页应用，支持 Hooks 特性 |
| **语言** | TypeScript | 5.x | 类型安全，提升代码质量和开发效率，便于大型项目维护 |
| **UI 组件库** | Ant Design | 5.x | 组件丰富，设计规范统一，支持主题定制和暗色模式 |
| **路由管理** | React Router | 6.x | React 生态标准路由方案，支持嵌套路由和路由守卫 |
| **状态管理** | Zustand | 4.x | 轻量级状态管理，使用简单，性能优秀 |
| **HTTP 客户端** | Axios | 1.x | 功能强大，支持拦截器、请求取消，社区成熟 |
| **代码高亮** | Prism.js | 1.x | 轻量级代码高亮库，支持多种语言，易于集成 |
| **图标库** | Lucide React | 0.x | 现代化图标库，支持 SVG 图标，可自定义颜色和大小 |
| **构建工具** | Vite | 6.x | 快速构建，热更新效率高，支持 React 开箱即用 |
| **CSS 工具** | Tailwind CSS | 3.x | 原子化 CSS，快速构建响应式布局，与 Ant Design 兼容 |

## 2. 技术选型详细说明

### 2.1 前端框架：React + TypeScript

**选型理由**：
- React 是目前最流行的前端框架，生态系统完善，社区资源丰富
- TypeScript 提供类型安全，在大型项目中能有效减少运行时错误
- Hooks 特性使状态管理和副作用处理更加简洁
- 与 Ant Design 组件库完美兼容

### 2.2 UI 组件库：Ant Design

**选型理由**：
- 提供丰富的高质量组件（表单、表格、Modal、Tabs 等）
- 支持主题定制和暗色模式切换
- 设计语言统一，符合现代 UI 设计标准
- 文档完善，社区活跃，问题解决速度快

### 2.3 路由管理：React Router

**选型理由**：
- React 官方推荐的路由方案，API 设计优雅
- 支持嵌套路由、路由参数、路由守卫等功能
- 与 React 18 完全兼容

### 2.4 状态管理：Zustand

**选型理由**：
- 轻量级状态管理，无需 Provider 嵌套
- API 简洁直观，学习成本低
- 性能优秀，支持中间件和持久化
- 适合本项目中等复杂度的状态管理需求（主题、语言、Token、收藏等）

### 2.5 HTTP 客户端：Axios

**选型理由**：
- 支持 Promise API，使用方便
- 支持请求和响应拦截器，便于统一处理 Token 和错误
- 支持取消请求、自动转换 JSON 数据
- 社区成熟，文档完善

### 2.6 代码高亮：Prism.js

**选型理由**：
- 轻量级，加载速度快
- 支持多种编程语言语法高亮（cURL、JavaScript、Python、Java、Golang 等）
- 支持行号显示和代码复制功能
- 主题丰富，可自定义样式

### 2.7 图标库：Lucide React

**选型理由**：
- 现代化图标设计，风格统一
- 支持 SVG 格式，可缩放不失真
- 支持自定义颜色、大小和 stroke width
- 包体积小，按需引入

### 2.8 构建工具：Vite

**选型理由**：
- 基于 ES Module，启动速度极快
- 热更新效率高，开发体验流畅
- 原生支持 TypeScript 和 React
- 配置简单，开箱即用

### 2.9 CSS 工具：Tailwind CSS

**选型理由**：
- 原子化 CSS，快速构建响应式布局
- 与 Ant Design 兼容良好
- 支持自定义主题配置
- 减少 CSS 文件体积，提升加载性能

## 3. 关键技术方案

### 3.1 OpenAPI 文档解析

**方案**：使用 `@apidevtools/swagger-parser` 库解析 OpenAPI 文档

**理由**：
- 支持 OpenAPI 2.0 和 3.0 规范
- 提供完整的类型定义
- 自动验证文档格式正确性

### 3.2 代码生成

**方案**：使用 `openapi-codegen` 或自定义模板生成代码

**支持语言**：
- cURL
- JavaScript (Fetch/axios)
- Python (requests)
- Java (OkHttp)
- Golang (net/http)

### 3.3 本地存储

**方案**：使用 `localStorage` 配合 Zustand 持久化中间件

**存储内容**：
- 收藏的接口列表
- 请求历史记录
- 全局 Token 设置
- 主题偏好
- 语言偏好

### 3.4 响应式布局

**方案**：Tailwind CSS + Ant Design 响应式组件

**断点设置**：
- 移动端：< 768px
- 平板：768px - 1200px
- 桌面端：> 1200px

## 4. 技术栈对比

| 对比维度 | 推荐方案 | 备选方案 | 选择理由 |
|---------|---------|---------|---------|
| 框架 | React | Vue | React 生态更成熟，组件库选择更多 |
| UI 库 | Ant Design | Material UI | Ant Design 中文文档完善，设计风格更符合参考界面 |
| 状态管理 | Zustand | Redux | Zustand 更轻量，学习成本低 |
| 构建工具 | Vite | Webpack | Vite 启动更快，开发体验更好 |
| CSS | Tailwind CSS | SCSS | Tailwind 更适合快速构建响应式布局 |

## 6. 注意事项

1. **Ant Design 5.x 与 Tailwind CSS 兼容**：需要配置 Tailwind 以避免样式冲突
2. **Prism.js 主题**：选择与 Ant Design 主题协调的代码高亮主题
3. **OpenAPI 文档兼容性**：需要处理不同版本的 OpenAPI 规范差异
4. **性能优化**：对于大型 OpenAPI 文档，考虑虚拟滚动和懒加载
